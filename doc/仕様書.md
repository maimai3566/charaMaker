# Chara-Maker (キャラメーカー) 仕様書

## 1. 概要

Chara-Makerは、Google Gemini 3.1 Flash Image Previewモデルを活用し、ゲーム等の開発で利用可能な「キャラクターの4ポーズ・スプライトシート」および「高精度な背景画像」を自動生成するWebアプリケーションです。

生成した画像は背景透過処理（キャラクターのみ）やWebP変換を行い、Firebase (Firestore / Storage) へ保存・管理できます。背景生成では多種多様なスタイルや雰囲気を指定可能です。

## 2. 技術スタック

- **フロントエンド:** Next.js (App Router), React, Tailwind CSS
- **バックエンド:** Next.js API Routes (Route Handlers)
- **AIモデル:**
  - 画像生成: Gemini 3.1 Flash Image Preview API (`gemini-3.1-flash-image-preview`)
  - テキスト支援: Gemini 2.5 Flash Lite (`gemini-2.5-flash-lite`)
- **データベース:** Firebase Firestore (lite)
- **ストレージ:** Firebase Storage

## 3. 主要機能

### 3.1 キャラクター生成

- ユーザーが入力したテキストプロンプトに基づき、Gemini APIを使用して画像を生成します。
- 生成される画像は4つのポーズが横に並んだスプライトシート形式です。
- アニメーションスタイルのプリセット（歩く、寝る、跳ねる等）を選択または直接指定できます。

### 3.2 画像プロセッシング

- **背景透過:** 白に近い色（RGBが240以上）を自動的に透過処理します。
- **リサイズ:** 生成された画像を 800x200px (1フレーム 200x200px × 4ポーズ) に最適化します。
- **軽量化:** 画像を WebP 形式に変換して保存・ダウンロード可能です。
- **アニメーションプレビュー:** スプライトシートから1フレームずつ切り出し、実際のアニメーションの動きをその場で確認できます。

### 3.3 マスターデータ管理 (CMS機能)

- **ゲームデータ登録:** 生成したキャラクターを選択し、モンスターのマスターデータとして Firestore に登録可能です。
- **マスターデータ一覧:** 登録済みの全データを一覧表示し、以下の操作が可能です。
  - **インライン編集:** ステータス（HP, ATK, DEF等）を一覧上で直接編集・更新。
  - **詳細編集:** 名前、説明、属性、進化段階などの全属性をフォームで修正。
  - **削除:** データの完全消去。
  - **並べ替え:** ID、名前、属性、ステータス各値での昇順・降順ソート。
- **重複登録防止:** すでにマスターデータとして登録済みの画像は、生成履歴から自動的に非表示になります。

### 3.4 AI支援機能

- **AI命名サポート:** キャラクターの属性やデザインコンセプトを元に、Gemini 2.5 Flash Lite が名前と説明文を提案します。
- **双方向翻訳機能:** 名前と説明文の内容を、ボタン一つで日本語⇔英語に翻訳（Geminiを使用）します。

### 3.5 インフラと履歴管理

- **Firebase連携:**
  - 画像本体 (.webp) は Firebase Storage に保存されます。
  - プロンプト、生成日時、画像URLなどのメタデータは Firestore に保存されます。
- **永続化:** 過去に生成したキャラクターを履歴として閲覧・再選択・削除可能です。

### 3.6 背景生成機能

- **スタイル指定 (Style):** 11のカテゴリーに分類された多種多様な画風・時代設定（大正ロマン、サイバーパンク、浮世絵、ファンタジー風等）を選択できます。選択すると、対応するキーワードが自動でプロンプトに組み込まれます。
- **雰囲気・ライティング (Atmosphere):** 「透明感」「退廃的」「シネマティック」などのプリセットから雰囲気を指定可能です。
- **条件の一致する参考画像表示:** 現在選択しているスタイルと雰囲気が一致する過去の生成画像を「参考画像」として自動表示し、完成イメージの参考にできます。
- **生成条件の再利用:** 履歴や参考画像から、当時の詳細プロンプト、スタイル、雰囲気、アスペクト比の設定をワンタップで復元（再現）できます。
- **アスペクト比選択:** 1:1, 4:3, 3:4, 16:9, 9:16, 4:1 の多様な比率に対応しています。
- **詳細情報の精緻な保存:** 生成時のプロンプト全体に加え、個別に選択したスタイル名、詳細記述、雰囲気のテキストを個別のフィールドとして Firestore に保存・管理します。

## 4. 画面構成

- **キャラクター生成タブ:**
  - **生成エリア:** プロンプト入力、アニメーションスタイル選択、スプライト生成。
  - **履歴エリア:** 生成済み画像の一覧。ここから「マスター登録へ」進むことができます。
- **背景生成タブ:**
  - **スタイル/雰囲気選択:** 階層型メニューからのスタイル選択、キーワードによる雰囲気指定。
  - **生成・参考エリア:** 背景生成ボタン、同一条件の過去画像（参考画像）の表示。
  - **背景履歴:** 保存済み背景の一覧。再利用・削除が可能。
- **ゲームデータ作成タブ:**
  - **登録・編集フォーム:** ID、名前（日・英）、属性、進化段階、ステータス、進化ルートの入力。AI支援ボタン（命名・翻訳）を配置。
  - **マスターデータ一覧:** 登録済みモンスターのテーブル。ソート、インライン編集、編集/削除ボタン。
