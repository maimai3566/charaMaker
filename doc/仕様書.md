# Chara-Maker (キャラメーカー) 仕様書

## 1. 概要
Chara-Makerは、Google Gemini 3.1 Flash Image Previewモデルを活用し、ゲーム等の開発で利用可能な「キャラクターの4コマ（4ポーズ）スプライトシート」を自動生成するWebアプリケーションです。

## 2. 技術スタック
- **フレームワーク:** Next.js (App Router), React
- **スタイリング:** Tailwind CSS
- **AIモデル:** Gemini 3.1 Flash Image Preview API (モデル名: `gemini-3.1-flash-image-preview`)
- **SDK:** `@google/genai`
- **データ永続化:** Web Storage API (`localStorage`)

## 3. 主要機能

### 3.1 プロンプト構築と画像生成 (`src/app/page.tsx`)
- ユーザーは「デザインの詳細（Character Prompt）」と「アニメーションの動き（Animation Style）」を入力・選択できます。
- **アニメーションプリセット:** 「歩く」「寝る」「うなずく」「呼吸する」「震える」など、複数パターンの動きをボタンから簡単に入力可能です。
- **プロンプトエンジニアリング:** 入力されたテキストに対し、ツール側で以下の「システム制約」を自動で付与し、理想的なスプライトシートが出力されるよう調整しています。
  - `exactly four evenly spaced horizontal poses` (等間隔の4ポーズ)
  - `SMALL and CENTERED ... absolutely no cropping` (見切れ防止のため中心に小さく描画)
  - `Do NOT draw boxes, borders, or panels` (コマ割りや枠線の排除)
  - `Solid white background, clear THICK black outer lines` (白背景、キャラクターのみ太い黒の輪郭線)

### 3.2 画像の表示とクロップ（切り出し）・リサイズ処理 (`ImageProcessor`)
- 生成された画像（Base64形式）を受け取り、画面上に「スプライトシート構築」結果として表示します。
- **4コマの切り出しとスケーリング処理:**
  - AIが生成した画像（通常は16:9等）を幅（Width）で単純に4等分します。
  - 各等分されたセクション（1コマ分）を、ゲーム用の等間隔な枠（200px × 200px）に配置します。
  - この際、キャラクターが見切れてしまわないように、セクションの縦横比を維持したまま `200x200` の枠内にぴったり収まるようスケールダウン（縮小処理）を行い、枠の中央に配置（レターボックス・ピラーボックス化）する仕組みになっています。

### 3.3 生成履歴のローカル保存 (`src/data/repository/LocalHistoryRepository.ts`)
- 生成に成功した画像とプロンプトは、自動的にローカルの `localStorage` に保存されます。
- 表示時には、新しいものが上に表示されるようソート（作成日時の降順）されます。
- **セーフガード機能（容量オーバー対策）:**
  - ブラウザの `localStorage` の一般的な容量上限（約5MB）に対処するため、履歴データの上限を**15件**に制限。
  - Base64の画像文字列が大きく、15件未満で容量フル（`QuotaExceededError`）になった場合は、最も古い履歴から1件ずつ順次`shift()`して削除し、保存可能になるまでリトライする仕組みを実装しています。

## 4. APIエンドポイント仕様

### `POST /api/generate` (`src/app/api/generate/route.ts`)
- **概要:** フロントエンドからのプロンプトを受け取り、Gemini APIを叩いて画像を生成します。
- **リクエストボディ:**
  ```json
  {
    "prompt": "フロントエンドで組み立てられた最終プロンプトの文字列"
  }
  ```
- **Gemini API 呼び出し設定 (`config`):**
  - `aspectRatio: "4:1"`: 4コマが横に綺麗に並ぶよう、アスペクト比を横長に指定。
  - `imageSize: "512"`: 生成サイズを制御。
  - `responseModalities: ["IMAGE"]`: 画像でのレスポンスを強制。
- **レスポンス:**
  - 成功: `200 OK`
    ```json
    { "imageBase64": "iVBORw0KGgoAAAANSUhEUg..." }
    ```
  - 失敗: `400 Bad Request` または `500 Internal Server Error`
    ```json
    { "error": "Prompt is required" }
    ```

## 5. 今後の展望・保守
- 現在は `localStorage` に画像をBase64で直接保持しているため、デバイス間の履歴共有はできません。本格的に履歴を長期間・大量に管理したい場合は、Firebase Storage などのクラウドストレージへの画像アップロードおよび Firestore でのメタデータ管理への移行を検討してください。
